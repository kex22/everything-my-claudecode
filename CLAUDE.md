# everything-my-claudecode

自用 Claude Code 插件集合。

## 技能编写经验

### 输出锚点防跳步

多步骤技能中，AI 容易把多个步骤合并成一篇"综合回复"而跳过中间步骤。尤其当某个步骤是开放式叙述（如"用自己的话介绍"）时，模型会从"逐步执行"模式滑入"自由写作"模式，连带后续步骤一起被吞掉。

**解法：给每个步骤加显式输出锚点。**

- 结构化步骤（检查清单、读文件报告）：在步骤开头加 `**先输出 "### 标题"，再开始执行。**`，强制模型在该时间点产生输出，打断"攒着一起写"的倾向
- 开放式步骤（介绍、说明）：提供输出模板（加粗小节标题），把"作文题"变成"填空题"——模板约束结构，内容仍然自由

**反例（容易被跳过）：**
```markdown
向用户介绍 docs/ 下三个目录的作用，用自己的话，结合项目实际。
```

**正例（稳定输出）：**
```markdown
必须输出以下三个小节（内容用自己的话，结合项目实际）：

**`docs/design/` — 设计方案（活文档）**
要点：...

**`docs/plans/` — 实施计划（执行清单）**
要点：...
```

### 前置准备分离读取与输出

步骤中混合"读文件"和"输出内容"两个动作时，模型读完文件后容易忘记输出（注意力被文件内容冲淡）。

**解法：加"前置准备"阶段，集中完成所有文件读取，正式步骤只负责消费已有信息并输出。**

```markdown
## 前置准备

1. 读取 xxx 了解规范
2. 读取 yyy 了解现状

准备完成后，依次执行以下步骤：

## 1. 输出 xxx（此时不再需要读文件，只管输出）
```
